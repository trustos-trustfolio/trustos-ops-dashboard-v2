# Contributing to Trust OS

Thank you for your interest in contributing.

## What We Welcome

- Bug reports with clear reproduction steps
- Documentation improvements and corrections
- SDK usage examples and integration patterns
- Feature requests and design discussions
- Pull requests for examples, docs, and SDK code

## What We Do Not Accept

- Pull requests that include API keys, secrets, or credentials
- Changes that hardcode or expose internal infrastructure
- Modifications to production API endpoints without prior discussion
- Issues or PRs that contain real API keys or access tokens — use placeholder values

## How to Contribute

1. **Open an issue first** — describe the bug or proposed change before writing code
2. **Fork the repository** and create a feature branch
3. **Make your changes** — no API key is required to work on examples or docs
4. **Test your changes** locally
5. **Submit a pull request** with a clear description

## Security

- Never include API keys, secrets, or credentials in issues, PRs, or commits
- Use `YOUR_API_KEY` or `process.env.TRUST_OS_API_KEY` as placeholders in all examples
- Report security vulnerabilities **privately** to founder@trust-os.io
- Do not open a public issue for security vulnerabilities

## Getting Help

- Open an issue for questions or discussions
- Email: admin@trust-os.io
- Website: [trust-os.io](https://trust-os.io)
