# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Trust OS, please report it **privately**.

**Email:** founder@trust-os.io

Do **not** open a public GitHub issue for security vulnerabilities. Public issues are visible to everyone before a fix is in place.

## What to Include

- A clear description of the vulnerability
- Steps to reproduce
- Potential impact and affected versions

We aim to respond within 72 hours and will coordinate a fix and responsible disclosure timeline with you.

## Security Rules for Contributors

- **Never include API keys, secrets, or credentials** in issues, pull requests, comments, or commits
- **Do not expose real API keys** in bug reports — use placeholder values like `YOUR_API_KEY`
- **Use environment variables** for all credentials — never hardcode keys in source files
- **Do not expose internal infrastructure** details, hostnames, or service URLs in public forks or PRs
- **Do not write sensitive information** in public issue titles or descriptions — email us instead
